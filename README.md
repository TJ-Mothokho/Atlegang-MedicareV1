# 🏥 Atlegang Medicare

> **Legacy University Project | Modernization in Progress**

## Overview

Atlegang Medicare is a hospital management system originally developed as a **third-year university group project**. The project was successfully completed and received a **final mark of 78% (Distinction)**.

This repository preserves the original implementation while serving as the foundation for a complete modernization effort. The goal is not only to improve the application but also to demonstrate modern software engineering practices and enterprise-level architecture.

## Why Revamp the Project?

Although the original project achieved its academic objectives, software development has evolved considerably. This revamp provides an opportunity to redesign the system using current industry standards, making it more scalable, maintainable, secure, and cloud-ready.

The project will gradually transition from a traditional layered architecture to a production-oriented solution suitable for real-world deployment.

## Modernization Goals

The new version aims to incorporate:

* Clean Architecture
* Domain-Driven Design (where appropriate)
* SOLID principles
* Repository and Unit of Work patterns
* CQRS (where beneficial)
* Dependency Injection
* Entity Framework Core best practices
* ASP.NET Core Web API
* JWT Authentication & Authorization
* API Versioning
* FluentValidation
* Global Exception Handling
* Structured Logging
* Unit & Integration Testing
* Docker containerization
* CI/CD pipelines
* Azure cloud services
* Azure SQL Database
* Azure App Service
* Azure Storage
* Azure Key Vault
* Azure Monitor & Application Insights
* Infrastructure as Code (where applicable)
* Comprehensive documentation

## Purpose

This repository is intended to showcase the transformation of an academic project into a modern, enterprise-style application. It serves as both a learning journey and a portfolio project, highlighting software architecture, cloud technologies, DevOps practices, and clean coding principles.

## Project Status

🚧 **Work in Progress**

The modernization will be completed incrementally. Features and architectural improvements will be introduced over time, with documentation updated throughout the process.
